1. 项目目标理解
--------------------------------------------------------------------------------
一款 Windows 本地桌面工具，核心功能是管理 Windows 文件夹的 ICO 图标。
通过修改文件夹内的 desktop.ini 来设置/恢复自定义图标，后续阶段扩展图标库
管理、收藏分组和拖拽操作。

设计原则：简单、高效、高颜值，不做无关功能，不过度设计。
目标平台：Windows 10 / 11。

2. 推荐技术栈
--------------------------------------------------------------------------------
桌面框架 : Tauri v2 (轻量, Rust 后端直接调用 Windows API)
前端     : React + TypeScript (组件化, 类型安全)
后端     : Rust (desktop.ini 读写, Explorer 刷新等系统操作)
包管理   : pnpm (前端) / Cargo (Rust)
UI 组件  : 原生 CSS / CSS Modules (阶段1 UI 简单, 不引入组件库避免过度设计)
构建输出 : MSI 安装版 + 便携版

3. 阶段1 MVP 功能范围
--------------------------------------------------------------------------------
仅实现 4 个功能:

  (1) 选择目标文件夹  — 系统文件夹选择对话框
  (2) 选择 ICO 文件   — 文件选择器过滤仅 *.ico
  (3) 应用图标       — 写入 desktop.ini (引用源 ICO 路径, 禁止复制)
  (4) 恢复默认图标    — 移除 IconResource 配置, 保留 desktop.ini
  (5) 刷新 Explorer  — 图标变更立即生效

明确禁止在阶段1实现:
  图标库、收藏夹、分组、拖拽、UI高级美化

6. desktop.ini 实现方案
--------------------------------------------------------------------------------
核心原则: 引用源 ICO 路径, 不复制 ICO 文件, 不破坏已有的非图标配置。

写入图标流程:
  1. Rust 端接收目标文件夹路径 + ICO 文件绝对路径
  2. 读取 {folder}\desktop.ini (不存在则创建)
  3. 设置文件属性为 SYSTEM | HIDDEN
  4. 设置文件夹属性包含 FILE_ATTRIBUTE_SYSTEM
  5. 解析现有 INI 内容 (按 section 拆分)
  6. 在 [.ShellClassInfo] section 下设置 IconResource={ico_path},0
  7. 保留所有其他 section 和 key 不变
  8. 写回文件

恢复默认图标流程:
  1. 读取 desktop.ini
  2. 仅移除 [.ShellClassInfo] 下的 IconResource 行
  3. 保留所有其他内容
  4. 即使 desktop.ini 全部为空, 也保留该文件 (不删除)

7. Explorer 刷新方案 (推荐方案 B)
--------------------------------------------------------------------------------
方案 A : SHChangeNotify(SHCNE_UPDATEITEM, ...) — 精准刷新单个文件夹
方案 B : SHChangeNotify(SHCNE_ASSOCCHANGED, ...) + IE COM 接口 — 全局刷新 [推荐]
方案 C : 重启 explorer.exe — 100% 生效但体验差

实现: Rust 调用 shell32.dll 的 SHChangeNotify, 暴露 refresh_explorer
命令给前端, 在图标操作成功后自动调用。多级策略: 先单文件夹, 不生效则全局。

8. 主要风险点
--------------------------------------------------------------------------------
🔴 高风险 (已解决):
  - desktop.ini 编码问题 → 研究结论: Windows INI API (GetPrivateProfileStringW)
    仅识别 ANSI(A) 和 UTF-16 LE with BOM 两种编码。UTF-8 不被支持，即使带 BOM。
    实现方案: 读取时自动检测 BOM (0xFF 0xFE → UTF-16 LE, 否则 ANSI)；
    写入时始终使用 UTF-16 LE with BOM 以确保最佳兼容性。
    注意: 不使用 Windows INI API，采用手动文件读写 + 自己的 INI 解析器。

🟡 中风险:
  - Explorer 刷新不生效: 某些 Windows 版本 SHChangeNotify 不可靠。
    缓解: 多级刷新策略。
  - 文件夹属性设置: 忘记设 System 属性导致 desktop.ini 被忽略。
    缓解: 封装原子操作。
  - Tauri v2 Windows 环境: VS Build Tools / Windows SDK 配置复杂。
    缓解: 环境配置文档先行。

🟢 低风险:
  - desktop.ini 并发写入 (极低概率)
  - 权限不足 (写前检查 + 友好提示)
  - 前端与 Rust 状态同步 (单向数据流)
