3. 关键技术决策
--------------------------------------------------------------------------------

  3.1 desktop.ini 编码
      读取: BOM 检测 (0xFF 0xFE) + GBK ANSI 回退 + UTF-8/windows-1252 兜底
      写入: 统一 UTF-16 LE + BOM (Windows INI API 兼容)
      依赖: encoding_rs = "0.8"

  3.2 Section 感知解析
      parse_icon_resource 仅处理 [.ShellClassInfo] 下的 IconResource
      其他 section 同名 key 完整保留

  3.3 ICO 引用策略
      源 ICO 路径引用，不复制 ICO 文件到目标文件夹

  3.4 恢复默认图标策略
      仅移除本工具写入的 IconResource，保留 desktop.ini 及其他配置
      即使 desktop.ini 为空也不删除文件

  3.5 Explorer 刷新
      二级策略: SHCNE_UPDATEDIR (精准) + SHCNE_ASSOCCHANGED (全局)

5. 已知限制与说明
--------------------------------------------------------------------------------

  5.1 cargo test 环境问题
      原因: Tauri 测试二进制链接 WebView2，当前环境缺少运行时入口点
            (STATUS_ENTRYPOINT_NOT_FOUND)。编译本身正常，仅运行失败。
      影响: 无法在 Tauri 项目内直接运行 cargo test
      缓解: 通过独立测试项目验证 desktop.ini 逻辑 (11/11 通过)
      注意: 不影响实际应用运行，仅在开发环境中影响测试执行

  5.2 构建环境依赖
      - Rust 1.96.0 (stable-x86_64-pc-windows-gnu)
      - MinGW-w64 GCC 15.2.0 (MSYS2)
      - Node.js v25.2.1 / pnpm 11.5.2
      - target-dir 重定向至 AppData\Local (绕过 OneDrive 安全软件)
