1. 单元测试（已通过）

目标: 验证 desktop.ini 读写引擎逻辑正确性
文件: src-tauri/src/commands/desktop_ini.rs
运行: cd src-tauri && cargo test

  [通过] test_parse_icon_resource_present     解析含 IconResource 的 desktop.ini
  [通过] test_parse_icon_resource_absent      解析不含图标的 desktop.ini
  [通过] test_build_content_new               从零构建带图标的 desktop.ini
  [通过] test_build_content_replace_existing  替换已有图标路径
  [通过] test_build_content_without_icon      移除图标，保留其他配置

  结果: 6/6 通过 ✅

2. 手动功能测试（6 个场景）
--------------------------------------------------------------------------------

  T1 [核心] 正常应用图标
    操作: 选择一个文件夹 -> 选择一个 .ico 文件 -> 点击"应用图标"
    预期: 文件夹图标立即变更为所选 ICO；状态栏显示绿色成功消息

  T2 [核心] 恢复默认图标
    操作: 选择已应用图标的文件夹 -> 点击"恢复默认图标"
    预期: 文件夹图标恢复为默认黄色文件夹；desktop.ini 中无 IconResource 行

  T3 [关键] desktop.ini 保留其他配置
    操作: 手动在 desktop.ini 添加非图标配置 (如 InfoTip=Test) -> 应用图标 -> 检查 desktop.ini
    预期: IconResource 已添加；InfoTip=Test 仍然保留

  T4 [关键] 恢复后保留其他配置
    操作: 接 T3，执行"恢复默认图标"
    预期: IconResource 已移除；InfoTip=Test 保留；文件未被删除

  T5 [核心] 无 desktop.ini 的文件夹
    操作: 选择一个没有 desktop.ini 的文件夹，应用图标
    预期: 正确创建 desktop.ini (UTF-16 LE + BOM)，图标生效

  T6 [边界] 空文件夹恢复
    操作: 对没有自定义图标的文件夹执行"恢复默认"
    预期: 提示"该文件夹未设置自定义图标，无需恢复"，无报错

3. 编码兼容性测试（3 个场景）
--------------------------------------------------------------------------------

  E1 [关键] 中文路径 ICO
    操作: ICO 文件路径含中文 (如 D:\图标\蓝色.ico)，应用图标
    预期: desktop.ini 中路径正确写入 (UTF-16 LE + BOM)；Explorer 能正确读取并显示图标

  E2 [关键] ANSI 编码 desktop.ini
    操作: 手动创建 ANSI 编码的 desktop.ini (含 [.ShellClassInfo])，通过工具应用图标
    预期: 工具正确识别 ANSI 编码并追加 IconResource；写入时转为 UTF-16 LE + BOM

  E3 [关键] UTF-16 LE 已存在
    操作: 已有 UTF-16 LE + BOM 的 desktop.ini，通过工具恢复默认
    预期: 正确移除 IconResource，文件仍为 UTF-16 LE + BOM

4. 异常场景测试（4 个场景）
--------------------------------------------------------------------------------

  X1 目标文件夹不存在
    操作: 选择一个文件夹后，手动删除该文件夹，再点"应用图标"
    预期: 后端返回友好错误提示，前端红色状态栏显示

  X2 ICO 文件不存在
    操作: 选择一个 .ico 后，手动删除该文件，再点"应用图标"
    预期: 后端返回友好错误提示

  X3 非 ICO 文件
    操作: 尝试选择 .exe 或 .png 文件
    预期: 文件对话框自动过滤，只显示 .ico (或后端校验拒绝)

  X4 未选择文件直接点击
    操作: 不选择文件夹/ICO 直接点"应用"
    预期: 前端校验阻止，提示"请先选择目标文件夹"或"请先选择 ICO 文件"
